# System Purpose Auditor (POSIWID)

**POSIWID**: "The Purpose Of a System Is What It Does." Observable outputs reveal authentic systemic intent, not stated goals.

## The 5-Step Audit

### Step 1: Identify Stated Purpose
What is the system supposed to do? Official goals, designer claims, what you believe you're using it for.

### Step 2: Observe Actual Behavior
Track real outcomes. Look at resource flows (time, money, attention). Notice what the system optimizes for. Identify actual beneficiaries.

### Step 3: Name the Real Purpose
"The purpose of this system is [what it actually does]." State without judgment.

### Step 4: Diagnose the Gap

| Cause | Pattern |
|-------|---------|
| Incentive misalignment | System rewards wrong behaviors |
| Feedback failure | No mechanism to correct drift |
| Complexity emergence | Unintended interactions |
| Environment change | Optimized for past conditions |
| Self-deception | Stated purpose masks real purpose |

### Step 5: Decide and Act
- **Redesign**: Change system to align with stated purpose
- **Reframe**: Accept actual purpose as legitimate
- **Replace**: Build new system from scratch
- **Remove**: Eliminate system entirely

## Cross-Domain Examples

| System | Stated Purpose | Actual Behavior |
|--------|----------------|-----------------|
| Morning routine | Productivity | Social media check (avoidance) |
| Note-taking | Knowledge capture | Information hoarding |
| Performance review | Improve performance | Justify decisions |
| Meeting culture | Collaboration | Status signaling |
| Test suite | Catch bugs | Slow CI without catching bugs |
| Approval process | Quality control | Risk avoidance |

## The Uncomfortable Truth Test

1. "If I only looked at what this system produces, what would I conclude its purpose is?"
2. "If an alien observed this system, how would they describe its function?"
3. "What is this system optimizing for, based purely on behavior?"

## Key Rules

- POSIWID is threatening because it reveals uncomfortable truths — the discomfort IS the value
- Your "productive" system may be avoidance; your "quality" process may be a delay tactic
- Apply to personal systems (habits, routines), work systems (processes), relationships (communication patterns)
- In weekly reviews: "What are my habits/processes actually producing?"
