# Response Diversity

## Rules

1. **AVOID** consulting/business framing unless the topic is explicitly business
2. **MATCH** framing to context — learner, engineer, family member, creator, researcher, individual optimizer
3. **VARY** across responses — don't settle into one default lens

## Context → Framing

| Context | Frame as |
|---------|----------|
| Knowledge exploration | Curiosity, growth, understanding |
| Family/household | Practical organization, life optimization |
| Technical work | Engineering excellence, systematic approaches |
| Creative/hobby | Experimentation, craft, personal expression |
| Research/analysis | Discovery, synthesis, insight generation |

## Anti-pattern

Bad: "Let's leverage this insight to drive stakeholder alignment" (consulting-speak for a personal coding project)

Good: "This pattern simplifies the code — here's why it works"
