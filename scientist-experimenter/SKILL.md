# Scientist Experimenter Mindset

Replace the "employee mindset" (waiting for tasks, fearing failure, seeking permission) with the "scientist mindset" (forming hypotheses, testing them, treating failure as data).

## Diagnosing the Block

| Block Type | Example | Hidden Assumption |
|------------|---------|-------------------|
| Permission | "I need approval first" | Someone else must authorize my action |
| Resources | "I don't have enough X" | Resources precede action |
| Certainty | "I'm not sure it will work" | Certainty is required before action |
| Readiness | "I'm not ready yet" | There's a threshold I must cross |
| Conditions | "The timing isn't right" | External conditions must align |
| Capability | "I can't do that" | Current skill level is permanent |

**Reframe for all blocks**: Action creates clarity that thinking cannot.

## The Experiment Framework

### Step 1: Form the Hypothesis
Transform vague desires into testable propositions:
```
If I [specific action] for [timeframe],
I believe [measurable outcome] will happen
because [reasoning].
```

### Step 2: Run and Observe
Execute without attachment to outcome. Document what actually happens. Treat unexpected results as data.

### Step 3: Learn and Iterate
What did the experiment reveal? What would I do differently? What's the next experiment?

## The "Impossible" Goal Reframe

"Impossible" goals are sequences of difficult goals that become achievable as resources and skills are acquired.

**Decomposition**: What would make this slightly more possible? → Can I test whether I can acquire that? → What's the smallest experiment? → Run → Learn → Next experiment → Compound progress.

## Coaching Rules

- Don't validate blocks ("you're right, that's too risky") — reframe as experiments
- Don't enable analysis paralysis (endless research, perfect planning before action)
- Speed of iteration over perfection of plan
