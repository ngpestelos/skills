# Iterative Refinement Workflow

Execute, measure, refine, repeat. Systematic improvement through continuous feedback loops.

## The 4-Step Cycle

### Step 1: Execute Minimum Viable Version

Start with the simplest version that:
- Produces measurable outcome
- Gets feedback quickly (not perfect)
- Is shippable in current state

"Don't wait for complete clarity — execution generates information planning cannot."

### Step 2: Measure Against Criteria

Define success criteria **before** execution:
- **Primary metric**: What defines success?
- **Failure indicators**: What signals problems?
- **Edge cases**: What boundary conditions matter?

Document results after execution — not just what was planned.

### Step 3: Identify Refinements

Based on measurements, prioritize:
1. **Fix failures** — what didn't work
2. **Reduce ambiguity** — clarify specifications
3. **Optimize** — improve what worked
4. **Expand** — add capabilities (only after core works)

### Step 4: Version and Repeat

Create next version with documented changes. Track learning in version history.

```
Version 1.0 → Measure → Refine → Version 1.1 → Measure → ...
```

## Meta-Review

After 3-5 iterations, analyze:
- Are iterations improving the primary metric consistently?
- Which refinement types yield biggest improvements?
- What failure patterns repeat?

**Exit when**: Diminishing returns — improvement < cost of continued iteration.
