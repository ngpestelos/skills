# Action-Clarity Loop

Generate clarity through action. Execute minimum viable actions to receive feedback that planning alone cannot provide.

## The Problem

| Planning Provides | Action Provides |
|-----------------|----------------|
| Theoretical models | Actual constraints |
| Predicted outcomes | Real feedback |
| Assumed requirements | Validated requirements |
| Known unknowns | Unknown unknowns |

## The 4-Step Loop

### Step 1: Diagnose the Blockage

What clarity do you feel you're missing? How long have you been in planning/analysis mode?

### Step 2: Define Minimum Viable Action

The smallest action that generates useful feedback:

- **Executable in <1 day**
- **Tests a key assumption**
- **Generates feedback** (not necessarily succeeds)
- **Reversible** if wrong

### Step 3: Execute and Capture Feedback

Before starting, document predictions. After executing, note:
- What actually happened (vs. predicted)
- Surprises and unexpected constraints
- What action revealed that planning couldn't

### Step 4: Refine and Repeat

Use feedback to update understanding. New clarity → next action. Loop until sufficient clarity to commit.

## Exit Criteria

**Stop when**:
- Sufficient clarity achieved to make informed decision
- Diminishing returns: action no longer revealing new information
- Risk acceptable within tolerance

**Continue when**:
- High uncertainty remains (many unknown unknowns)
- Low action cost (easy to run another test)
- Reversible decisions (can pivot without major cost)
