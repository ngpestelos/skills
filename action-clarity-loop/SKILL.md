# Action-Clarity Loop

Generate clarity through action. Planning gives theoretical models, predicted outcomes, assumed requirements, and known unknowns; action gives actual constraints, real feedback, validated requirements, and unknown unknowns.

## The 4-Step Loop

1. **Diagnose the blockage** — what clarity is missing, and how long has planning been running?
2. **Define minimum viable action** — smallest action that generates useful feedback. Executable in <1 day, tests a key assumption, generates feedback (not necessarily success), reversible if wrong.
3. **Execute and capture feedback** — document predictions before, note actual vs. predicted after, including surprises and constraints planning couldn't reveal.
4. **Refine and repeat** — update understanding, pick next action, loop until sufficient clarity to commit.

## Exit Criteria

**Stop** when clarity is sufficient for an informed decision, action no longer reveals new information, or risk is within tolerance.

**Continue** when uncertainty is high (many unknown unknowns), action cost is low, or decisions remain reversible.
